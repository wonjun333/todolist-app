## 테스트 환경 이슈

### React 19 + Playwright controlled input 호환성

Playwright의 `fill()` 및 `pressSequentially()` 메서드가 React 19 controlled input의 `onChange` 이벤트를 트리거하지 않는 문제가 확인되었습니다.

**원인:** React 19에서 입력 이벤트 처리 방식 변경  
**우회 방법:** `HTMLInputElement.prototype.value` native setter + `input`/`change` 이벤트 수동 dispatch

```javascript
const nativeSetter = Object.getOwnPropertyDescriptor(window.HTMLInputElement.prototype, 'value').set;
nativeSetter.call(input, 'value');
input.dispatchEvent(new Event('input', { bubbles: true }));
input.dispatchEvent(new Event('change', { bubbles: true }));
```

## 발견된 버그

### BUG-01: 로그인 실패 시 오류 메시지 미표시 (심각도: 중)

**시나리오:** SC-A-02 예외 시나리오  
**재현 방법:** 잘못된 이메일/비밀번호로 로그인 시도  
**기대 동작:** "이메일 또는 비밀번호가 올바르지 않습니다." 오류 메시지 표시  
**실제 동작:** 오류 메시지가 표시되지 않고 폼이 초기화됨  
**서버 응답:** HTTP 401 정상 반환 확인됨  
**추정 원인:** TanStack Query의 `mutate()` 옵션으로 전달된 `onError` 콜백이 React 19 렌더링 사이클과의 타이밍 이슈로 실행되지 않거나, `onChange` 핸들러의 `setError('')` 호출이 오류 상태를 초기화함

### BUG-02: 날짜 UTC 오프셋으로 인한 1일 오차 (심각도: 하)

**시나리오:** SC-T-01, SC-T-02, SC-F-03  
**재현 방법:** 종료예정일을 2026-05-15(KST)로 설정하면 2026-05-14로 표시됨  
**기대 동작:** 입력한 날짜 그대로 표시  
**실제 동작:** UTC 변환으로 인해 1일 이전 날짜로 표시  
**추정 원인:** 백엔드에서 날짜를 UTC로 저장하고, 프론트엔드에서 로컬 타임존 변환 없이 UTC 기준으로 표시
