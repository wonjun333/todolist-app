# 도메인 정의서

**프로젝트명:** TodoListApp  
**버전:** 1.0.0  
**작성일:** 2026-05-12  
**작성자:** Business Analyst

---

## 1. 프로젝트 개요

### 1.1 프로젝트명

TodoListApp — 인증 기반 개인 할일 관리 애플리케이션

### 1.2 목적

회원가입 및 로그인을 통해 인증된 사용자가 자신만의 할일을 데이터베이스에 안전하게 저장하고, 카테고리 및 다양한 조건으로 관리할 수 있는 개인화된 할일 관리 서비스를 제공한다.

### 1.3 해결하는 문제

| 문제 | 해결 방안 |
|------|-----------|
| 개인별 할일을 체계적으로 관리할 수 있는 전용 앱 부재 | 사용자 인증 기반의 개인화된 할일 관리 기능 제공 |
| 인증된 사용자 단위로 데이터를 분리 저장하는 앱 부재 | 사용자 계정과 연동된 독립적인 데이터 저장 구조 적용 |
| 카테고리 별로 할일을 분류하고 등록할 수 있는 앱 부재 | 기본 카테고리 및 사용자 정의 카테고리 등록 기능 제공 |

## 2. 핵심 도메인 개념

| 용어 | 영문명 | 정의 |
|------|--------|------|
| 사용자 | User | 회원가입을 통해 계정을 생성하고 인증 후 서비스를 이용하는 주체 |
| 할일 | Todo | 사용자가 등록하는 작업 단위. 제목, 설명, 종료예정일, 카테고리, 완료 여부를 포함 |
| 카테고리 | Category | 할일을 분류하기 위한 그룹. 기본 카테고리와 사용자 정의 카테고리로 구분 |
| 기본 카테고리 | Default Category | 시스템이 제공하는 사전 정의 카테고리 (예: 일반, 업무, 개인) |
| 사용자 정의 카테고리 | Custom Category | 인증된 사용자가 직접 생성한 카테고리 |
| 인증 | Authentication | 사용자가 이메일과 비밀번호로 신원을 확인하고 서비스 접근 권한을 획득하는 과정 |
| 완료 여부 | Completion Status | 할일의 처리 상태 (미완료 / 완료) |
| 종료예정일 | Due Date | 할일을 완료해야 하는 목표 날짜 |

## 3. 도메인 모델

### 3.1 핵심 엔티티 및 속성

#### User (사용자)

| 속성 | 타입 | 설명 | 제약 |
|------|------|------|------|
| id | UUID | 사용자 고유 식별자 | PK, 자동 생성 |
| email | String | 로그인 이메일 | Unique, Not Null |
| password | String | 암호화된 비밀번호 | Not Null |
| name | String | 사용자 이름 | Not Null |
| createdAt | DateTime | 계정 생성일시 | Not Null |
| updatedAt | DateTime | 정보 수정일시 | Not Null |

#### Todo (할일)

| 속성 | 타입 | 설명 | 제약 |
|------|------|------|------|
| id | UUID | 할일 고유 식별자 | PK, 자동 생성 |
| userId | UUID | 소유 사용자 식별자 | FK → User.id, Not Null |
| categoryId | UUID | 분류 카테고리 식별자 | FK → Category.id, Not Null |
| title | String | 할일 제목 | Not Null |
| description | String | 할일 설명 | Nullable |
| dueDate | Date | 종료예정일 | Nullable |
| isCompleted | Boolean | 완료 여부 | Default: false |
| createdAt | DateTime | 등록일시 | Not Null |
| updatedAt | DateTime | 수정일시 | Not Null |

#### Category (카테고리)

| 속성 | 타입 | 설명 | 제약 |
|------|------|------|------|
| id | UUID | 카테고리 고유 식별자 | PK, 자동 생성 |
| userId | UUID | 소유 사용자 식별자 | FK → User.id, Nullable (기본 카테고리는 NULL) |
| name | String | 카테고리명 | Not Null |
| isDefault | Boolean | 기본 카테고리 여부 | Default: false |
| createdAt | DateTime | 생성일시 | Not Null |

### 3.2 엔티티 관계

```
User ||--o{ Todo       : "소유한다 (1:N)"
User ||--o{ Category   : "생성한다 (1:N)"
Category ||--o{ Todo   : "분류한다 (1:N)"
```

- 한 명의 사용자는 여러 개의 할일을 소유할 수 있다.
- 한 명의 사용자는 여러 개의 사용자 정의 카테고리를 생성할 수 있다.
- 하나의 카테고리에는 여러 개의 할일이 분류될 수 있다.
- 기본 카테고리(isDefault = true)는 모든 사용자가 공통으로 사용 가능하며, userId는 NULL이다.
- 사용자 정의 카테고리(isDefault = false)는 해당 사용자에게만 귀속된다.

## 4. 핵심 비즈니스 규칙

### 4.1 사용자(User) 규칙

- **BR-U-01** 이메일은 시스템 전체에서 유일해야 한다. 중복 이메일로 회원가입할 수 없다.
- **BR-U-02** 비밀번호는 반드시 암호화(해시)하여 저장해야 하며, 평문으로 저장할 수 없다.
- **BR-U-03** 인증되지 않은 사용자는 할일 및 카테고리 관련 기능에 접근할 수 없다.
- **BR-U-04** 사용자는 자신의 정보(이름, 비밀번호)만 수정할 수 있으며, 타인의 정보는 조회하거나 수정할 수 없다.

### 4.2 할일(Todo) 규칙

- **BR-T-01** 할일은 반드시 인증된 사용자에게 귀속되어야 한다. 소유자 없는 할일은 존재할 수 없다.
- **BR-T-02** 할일 제목(title)은 필수 입력값이다.
- **BR-T-03** 할일에는 반드시 하나의 카테고리가 지정되어야 한다.
- **BR-T-04** 사용자는 자신의 할일만 조회, 수정, 삭제할 수 있다.
- **BR-T-05** 할일 생성 시 완료 여부(isCompleted)의 초기값은 미완료(false)이다.
- **BR-T-06** 종료예정일(dueDate)은 선택 항목이지만, 입력 시 유효한 날짜 형식이어야 한다.

### 4.3 카테고리(Category) 규칙

- **BR-C-01** 기본 카테고리는 시스템에서 관리하며, 사용자가 수정하거나 삭제할 수 없다.
- **BR-C-02** 사용자 정의 카테고리는 해당 사용자만 수정 및 삭제할 수 있다.
- **BR-C-03** 카테고리명은 동일 사용자 범위 내에서 중복될 수 없다.
- **BR-C-04** 할일이 연결된 카테고리는 삭제할 수 없다. 삭제 전 연결된 할일을 다른 카테고리로 이동하거나 삭제해야 한다.

## 5. 주요 유스케이스

### 5.1 인증 관련

| ID | 유스케이스 | 액터 | 간략 흐름 |
|----|-----------|------|-----------|
| UC-A-01 | 회원가입 | 비인증 사용자 | 이메일, 비밀번호, 이름 입력 → 중복 이메일 검증 → 계정 생성 |
| UC-A-02 | 로그인 | 비인증 사용자 | 이메일, 비밀번호 입력 → 자격증명 검증 → 인증 토큰 발급 |
| UC-A-03 | 로그아웃 | 인증된 사용자 | 인증 토큰 무효화 → 세션 종료 |

### 5.2 사용자 정보 관리

| ID | 유스케이스 | 액터 | 간략 흐름 |
|----|-----------|------|-----------|
| UC-U-01 | 개인정보 수정 | 인증된 사용자 | 수정할 항목 입력(이름, 비밀번호) → 유효성 검증 → 저장 |

### 5.3 할일 관리

| ID | 유스케이스 | 액터 | 간략 흐름 |
|----|-----------|------|-----------|
| UC-T-01 | 할일 등록 | 인증된 사용자 | 제목, 설명, 종료예정일, 카테고리 입력 → 유효성 검증 → 저장 |
| UC-T-02 | 할일 수정 | 인증된 사용자 | 대상 할일 선택 → 항목 수정 → 저장 |
| UC-T-03 | 할일 삭제 | 인증된 사용자 | 대상 할일 선택 → 소유권 확인 → 삭제 |
| UC-T-04 | 완료 처리 | 인증된 사용자 | 대상 할일 선택 → 완료 여부 토글 → 저장 |

### 5.4 카테고리 관리

| ID | 유스케이스 | 액터 | 간략 흐름 |
|----|-----------|------|-----------|
| UC-C-01 | 카테고리 추가 | 인증된 사용자 | 카테고리명 입력 → 중복 검증 → 사용자 정의 카테고리 생성 |
| UC-C-02 | 카테고리 수정 | 인증된 사용자 | 사용자 정의 카테고리 선택 → 이름 수정 → 저장 |
| UC-C-03 | 카테고리 삭제 | 인증된 사용자 | 사용자 정의 카테고리 선택 → 연결 할일 존재 여부 확인 → 삭제 |

### 5.5 조회 및 필터링

| ID | 유스케이스 | 액터 | 간략 흐름 |
|----|-----------|------|-----------|
| UC-F-01 | 할일 목록 조회 | 인증된 사용자 | 자신의 할일 전체 목록 조회 |
| UC-F-02 | 카테고리별 필터링 | 인증된 사용자 | 특정 카테고리 선택 → 해당 카테고리의 할일만 표시 |
| UC-F-03 | 종료 여부 기간 필터링 | 인증된 사용자 | 기간 조건 입력(시작일~종료일) → 종료예정일이 해당 기간에 속하는 할일 표시 |
| UC-F-04 | 완료 여부 필터링 | 인증된 사용자 | 완료/미완료 조건 선택 → 해당 상태의 할일만 표시 |
| UC-F-05 | 복합 필터링 | 인증된 사용자 | 카테고리, 기간, 완료 여부 조건 조합 → 조건에 부합하는 할일 목록 표시 |

## 6. 도메인 이벤트

| 이벤트명 | 트리거 조건 | 주요 후속 처리 |
|---------|------------|--------------|
| UserRegistered | 회원가입 성공 | 기본 카테고리 자동 할당, 환영 알림 |
| UserLoggedIn | 로그인 성공 | 인증 토큰 발급, 마지막 로그인 시각 갱신 |
| UserProfileUpdated | 개인정보 수정 성공 | updatedAt 갱신 |
| TodoCreated | 할일 등록 성공 | 할일 목록 갱신 |
| TodoUpdated | 할일 수정 성공 | 할일 목록 갱신, updatedAt 갱신 |
| TodoCompleted | 완료 처리 성공 | isCompleted = true 반영, 할일 목록 갱신 |
| TodoDeleted | 할일 삭제 성공 | 할일 목록에서 제거 |
| CategoryCreated | 사용자 정의 카테고리 생성 성공 | 카테고리 선택 목록 갱신 |
| CategoryUpdated | 카테고리 수정 성공 | 카테고리 선택 목록 및 연결된 할일 표시 갱신 |
| CategoryDeleted | 카테고리 삭제 성공 | 카테고리 선택 목록에서 제거 |
