## 1. 문서 개요

### 1.1 목적

본 문서는 TodoListApp v1의 6개 주요 화면에 대한 ASCII 와이어프레임을 제공한다. 디자인, 개발팀, QA가 레이아웃과 UI 구성을 일관되게 이해하고 구현하기 위한 기준을 제시한다.

### 1.2 스코프

- 데스크탑 기준 레이아웃 (너비 약 80자)
- 반응형 고려사항 (모바일/태블릿/데스크탑 브레이크포인트)
- 로딩, 빈 상태, 에러 상태 등 주요 변형 포함

### 1.3 표기법

| 표기 | 의미 |
|------|------|
| `+---+` | 박스 테두리 |
| `[ 버튼 ]` | 클릭 가능한 버튼 |
| `[________________]` | 입력 필드 |
| `[x] 항목` | 체크박스 (체크됨) |
| `[ ] 항목` | 체크박스 (미체크) |
| `[선택값        v]` | 드롭다운 |
| `(o) 선택` | 라디오 버튼 (선택됨) |
| `<태그명>` | 배지/태그 |
| `>> 링크` | 하이퍼링크 |
| `[* 활성 ]` | 활성 탭 |
| `! 에러 메시지` | 에러 상태 |
| `v 성공 메시지` | 성공 상태 |

## 2. 공통 레이아웃

### 2.1 로그인 전 페이지 (비인증)

```
================================================================================
                                TodoListApp
                            (App Logo/Title)
================================================================================

                    [본 페이지 콘텐츠]

                    >> 회원가입 또는 로그인 링크

================================================================================
```

### 2.2 로그인 후 헤더 (인증)

```
================================================================================
TodoListApp  [*Home] [Categories] [Profile]  User_Name  [ 로그아웃 ]
================================================================================
```

| 컴포넌트 | 타입 | 설명 |
|---------|------|------|
| TodoListApp | 로고/타이틀 | 앱 이름 클릭 시 목록 화면으로 이동 |
| Home 탭 | 네비게이션 | 현재 페이지 강조 표시 |
| Categories 탭 | 네비게이션 | 카테고리 관리 화면 이동 |
| Profile 탭 | 네비게이션 | 개인정보 수정 화면 이동 |
| User_Name | 텍스트 | 현재 로그인 사용자 이름 표시 |
| 로그아웃 버튼 | 버튼 | 로그아웃 처리 및 로그인 화면 이동 |

## 3. SCR-01 회원가입 화면

### 3.1 정상 상태

```
================================================================================
                              회원가입
================================================================================

이름
[________________]

이메일
[________________]

비밀번호 (8자 이상)
[________________]

비밀번호 확인
[________________]

                        [ 가입 ]  [ 로그인 페이지로 ]

================================================================================

이미 계정이 있으신가요? >> 로그인하기

================================================================================
```

| 컴포넌트 | 타입 | 설명 |
|---------|------|------|
| 이름 필드 | 텍스트 입력 | 사용자 이름 (필수) |
| 이메일 필드 | 이메일 입력 | 이메일 주소 (필수, 형식 검증) |
| 비밀번호 필드 | 비밀번호 입력 | 비밀번호 입력 (필수, 8자↑) |
| 비밀번호 확인 필드 | 비밀번호 입력 | 비밀번호 재입력 (필수, 일치 검증) |
| 가입 버튼 | 버튼 | 회원가입 요청 |
| 로그인 페이지로 버튼 | 링크 버튼 | SCR-02 이동 |
| 로그인하기 링크 | 링크 | SCR-02 이동 |

### 3.2 에러 상태

```
================================================================================
                              회원가입
================================================================================

이름
[________________]

이메일
[________________]
! 올바른 이메일 형식이 아닙니다.

비밀번호 (8자 이상)
[________________]

비밀번호 확인
[________________]
! 비밀번호가 일치하지 않습니다.

                   [ 가입 (비활성) ]  [ 로그인 페이지로 ]

================================================================================
```

### 3.3 서버 오류 상태 (이메일 중복)

```
================================================================================
                              회원가입
================================================================================

이메일
[________________]
! 이미 사용 중인 이메일입니다.

                        [ 가입 ]  [ 로그인 페이지로 ]

================================================================================
```

### 3.4 로딩 상태

```
================================================================================
                              회원가입
================================================================================

이메일
[________________]

비밀번호
[________________]

                  [ 가입 중... (로딩) ]  [ 로그인 페이지로 ]

================================================================================
```

### 3.5 성공 상태 (자동 리다이렉트)

```
================================================================================
v 가입이 완료되었습니다. 로그인 화면으로 이동합니다...
================================================================================
```

## 5. SCR-03 할일 목록 화면

### 5.1 정상 상태 (데이터 있음)

```
================================================================================

================================================================================

[ 필터 ]

[전체        v] [2026-05-01] ~ [2026-05-31] [전체 v] [ 필터 초기화 ]

────────────────────────────────────────────────────────────────────────────

할일 등록 [ ]

────────────────────────────────────────────────────────────────────────────

[x] 마케팅 기획안 제출
    <업무> | 2026-05-13
    [ 삭제 ]

[ ] 아침 운동 30분
    <개인> | 2026-05-13
    [ 삭제 ]

[ ] 팀 회의 준비
    <업무> | 2026-05-14
    [ 삭제 ]

[ ] 책 읽기 30페이지
    <개인> | 2026-05-15
    [ 삭제 ]

────────────────────────────────────────────────────────────────────────────
```

| 컴포넌트 | 타입 | 설명 |
|---------|------|------|
| 카테고리 필터 | 드롭다운 | "전체" / 기본 카테고리 / 사용자 정의 카테고리 |
| 기간 필터 | 날짜 입력 | 시작일 ~ 종료일 (선택사항) |
| 완료 여부 필터 | 라디오 | 전체 / 미완료 / 완료 |
| 필터 초기화 | 버튼 | 모든 필터 제거 |
| 할일 등록 | 버튼 | SCR-04 (등록 모드) 이동 |
| 할일 카드 | 카드 컴포넌트 | 제목, 카테고리 배지, 마감일, 완료 체크박스, 삭제 버튼 |
| 체크박스 | 체크박스 | 완료 여부 토글 (낙관적 업데이트) |
| 삭제 버튼 | 버튼 | 삭제 확인 다이얼로그 → 삭제 처리 |

## 6. SCR-04 할일 등록/수정 화면

### 6.1 등록 모드 (신규)

```
================================================================================
TodoListApp  [*Home] [Categories] [Profile]  Kim_Jisoo  [ 로그아웃 ]
================================================================================

                            할일 등록

────────────────────────────────────────────────────────────────────────────

제목 (필수)
[________________]

설명 (선택)
[_____________________________________________]
[_____________________________________________]

종료예정일 (선택)
[2026-05-13]

카테고리 (필수)
[일반         v]

────────────────────────────────────────────────────────────────────────────

                      [ 저장 ]  [ 취소 ]

================================================================================
```

| 컴포넌트 | 타입 | 설명 |
|---------|------|------|
| 제목 필드 | 텍스트 입력 | 할일 제목 (필수) |
| 설명 필드 | 텍스트 영역 | 상세 설명 (선택) |
| 종료예정일 필드 | 날짜 입력 | 마감일 (선택) |
| 카테고리 드롭다운 | 선택 목록 | 기본 + 사용자 정의 카테고리 (필수) |
| 저장 버튼 | 버튼 | 할일 생성/수정 후 목록 이동 |
| 취소 버튼 | 버튼 | 입력 미저장, 목록 이동 |

### 6.2 수정 모드 (기존 값 pre-fill)

```
================================================================================
TodoListApp  [*Home] [Categories] [Profile]  Kim_Jisoo  [ 로그아웃 ]
================================================================================

                            할일 수정

────────────────────────────────────────────────────────────────────────────

제목 (필수)
[팀 회의 준비________]

설명 (선택)
[회의실 B로 변경됨, 자료 준비 필요___________]

종료예정일 (선택)
[2026-05-14]

카테고리 (필수)
[업무          v]

────────────────────────────────────────────────────────────────────────────

                      [ 저장 ]  [ 취소 ]

================================================================================
```

### 6.3 검증 오류 상태

```
================================================================================

제목 (필수)
[________________]
! 제목은 필수 항목입니다.

────────────────────────────────────────────────────────────────────────────

```

### 6.4 로딩 상태

## 7. SCR-05 카테고리 관리 화면

### 7.1 정상 상태

```
================================================================================
TodoListApp  [Categories] [Home] [Profile]  Kim_Jisoo  [ 로그아웃 ]
================================================================================

                          카테고리 관리

────────────────────────────────────────────────────────────────────────────

새 카테고리 추가

[________________]  [ 추가 ]

────────────────────────────────────────────────────────────────────────────

기본 카테고리 (수정/삭제 불가)

* 일반              (기본)
* 업무              (기본)
* 개인              (기본)

────────────────────────────────────────────────────────────────────────────

사용자 정의 카테고리

* 사이드 프로젝트   [ 수정 ]  [ 삭제 ]
* 독서 & 학습       [ 수정 ]  [ 삭제 ]
* 2Q 프로젝트       [ 수정 ]  [ 삭제 ]

────────────────────────────────────────────────────────────────────────────
```

| 컴포넌트 | 타입 | 설명 |
|---------|------|------|
| 카테고리명 입력 필드 | 텍스트 입력 | 새 카테고리 이름 |
| 추가 버튼 | 버튼 | 새 카테고리 생성 |
| 기본 카테고리 목록 | 읽기전용 | 일반, 업무, 개인 (수정·삭제 불가 표시) |
| 사용자 정의 카테고리 목록 | 항목 | 이름, 수정/삭제 버튼 |
| 수정 버튼 | 버튼 | 인라인 수정 모드 진입 |
| 삭제 버튼 | 버튼 | 삭제 확인 다이얼로그 |

### 7.2 인라인 수정 모드

### 7.5 로딩 상태


## 9. 상태별 변형

### 9.1 로딩 상태 (공통)

모든 화면에서 API 호출 중일 때:
- 버튼 비활성화
- 로딩 인디케이터(스피너) 또는 텍스트 표시
- 입력 필드 비활성화 (선택적)

### 9.2 에러 상태 (공통)

| 에러 타입 | 메시지 | 처리 |
|----------|--------|------|
| 필수 필드 누락 | "{필드명}은(는) 필수 항목입니다." | 해당 필드 강조 + 에러 표시 |
| 형식 오류 | "올바른 {형식} 형식이 아닙니다." | 해당 필드 강조 + 에러 표시 |
| 중복/409 | "이미 사용 중인 {항목}입니다." | 필드 하단 에러 표시 |
| 접근 권한 없음/403 | "접근 권한이 없습니다." | 모달 다이얼로그 + 홈 이동 버튼 |
| 존재하지 않음/404 | "{항목}을(를) 찾을 수 없습니다." | 모달 다이얼로그 + 목록 이동 |
| 비인증/401 | "세션이 만료되었습니다. 다시 로그인해 주세요." | 자동 로그인 화면 리다이렉트 |
| 서버 에러/5xx | "요청을 처리할 수 없습니다. 잠시 후 다시 시도해 주세요." | 재시도 버튼 제공 |

### 9.3 빈 상태 (Empty State)

| 화면 | 메시지 | 액션 |
|------|--------|------|
| SCR-03 할일 목록 | "등록된 할일이 없습니다. 새 할일을 추가해 보세요." | [ 할일 등록 ] 버튼 |
| SCR-03 필터 결과 | "해당하는 할일이 없습니다. 필터 조건을 조정해 보세요." | [ 필터 초기화 ] 버튼 |
| SCR-05 사용자 카테고리 | (기본 카테고리만 표시) | 카테고리 추가 폼 표시 |

### 9.4 성공 상태

| 작업 | 메시지 | 동작 |
|------|--------|------|
| 회원가입 | "가입이 완료되었습니다." | 자동 로그인 화면 이동 (1~2초) |
| 할일 저장 | (메시지 없음, 목록 이동) | 목록 화면으로 자동 이동 |
| 카테고리 추가 | "카테고리가 추가되었습니다." | 토스트/배너 2~3초 표시 |
| 개인정보 저장 | "저장이 완료되었습니다." | 토스트/배너 2~3초 표시 |

### 10.1 기본 플로우

```
SCR-01 (회원가입)

    |
    +---> [ 할일 수정 ] ---> SCR-04 (수정 모드, 단건 조회 선행)
    |
    +---> [ 할일 삭제 ] ---> 확인 다이얼로그 --> 삭제 처리
    |
    +---> [ 할일 완료 ] ---> 체크박스 토글 (낙관적 업데이트)
    |
    +---> [ Categories 탭 ] ---> SCR-05 (카테고리 관리)
    |
    +---> [ Profile 탭 ] ---> SCR-06 (개인정보 수정)
    |
    +---> [ 로그아웃 ] ---> SCR-02 (로그인)

SCR-04 (할일 등록/수정)
    |
    +---> [ 저장 ] ---> SCR-03 (목록으로 복귀)
    |
    +---> [ 취소 ] ---> SCR-03 (저장 안함)

SCR-05 (카테고리 관리)
    |
    +---> [ 추가 ] ---> 카테고리 추가 (목록 갱신)
    |
    +---> [ 수정 ] ---> 인라인 수정
    |
    +---> [ 삭제 ] ---> 확인 다이얼로그 --> 삭제 처리
    |
    +---> [ Home 탭 ] ---> SCR-03

SCR-06 (개인정보 수정)
    |
    +---> [ 저장 ] ---> 저장 완료 (화면 유지)
    |
    +---> [ 취소 ] ---> SCR-03 (할일 목록)
```

### 10.2 인증 플로우

```
[비인증]
  |
  +---> 비인증 페이지 접근 가능 (SCR-01, SCR-02)
  |
  +---> 인증 필요 페이지 접근 시도 --> /auth/login 리다이렉트

[인증됨]
  |
  +---> 인증 필요 페이지 접근 가능 (SCR-03, SCR-04, SCR-05, SCR-06)
  |
  +---> SCR-01, SCR-02 접근 시도 --> /todos (목록) 리다이렉트
  |
  +---> 토큰 만료 감지 (401 응답) --> 자동 로그아웃 + /auth/login 리다이렉트
  |
  +---> 로그아웃 버튼 클릭 --> 토큰 삭제 + 상태 초기화 + /auth/login 이동
```

### 10.3 토큰 저장 방식 (보안)

```
로그인 성공
  |
  v
JWT 토큰 발급 (서버)
  |
  v
Zustand store 메모리 저장 (localStorage 미사용)
  |
  +---> HTTP 인터셉터에서 자동으로 Authorization 헤더 주입
  |
  +---> 페이지 새로고침 --> 토큰 초기화 (메모리 휘발)
  |                       --> 로그인 화면으로 리다이렉트
  |
  +---> 401 응답 (토큰 만료/무효) --> 자동 로그아웃
```
